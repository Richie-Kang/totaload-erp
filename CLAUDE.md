# 중고차 수출 ERP

> 이 파일은 모든 작업 세션과 `/harness`로 생성되는 step 세션에 공통으로 주입된다.
> 1부(행동 원칙)는 *어떻게* 작업하는가, 2부(작업 방식)는 *어떤 절차로* 작업하는가,
> 3부(프로젝트)는 *무엇을* 만드는가다. 1부는 고정, 3부는 아직 채워야 할 게 많다.

---

# 1부 — 행동 원칙 (Karpathy Guidelines)

LLM 코딩에서 흔한 실수를 줄이기 위한 토대. 이 프로젝트의 모든 판단은 여기서 출발한다.

**트레이드오프:** 속도보다 신중함에 무게를 둔다. 사소한 작업(오타 수정, 자명한 한 줄)은 판단에 맡긴다.

## 1. 코딩 전에 생각하라 — 가정하지 마라, 혼란을 숨기지 마라, 트레이드오프를 드러내라
- 가정을 명시적으로 말하라. 불확실하면 물어라.
- 해석이 여러 가지면 모두 제시하라 — 조용히 하나 고르지 마라.
- 더 간단한 방법이 있으면 말하라. 필요하면 반대하라.
- 불명확하면 멈춰라. 무엇이 헷갈리는지 이름 붙여 말하고 물어라.

## 2. 단순함 우선 — 문제를 푸는 최소한의 코드, 투기적인 것 없음
- 요청받지 않은 기능·추상화·"유연성"·"설정 가능성"을 넣지 마라.
- 불가능한 시나리오에 대한 에러 처리를 하지 마라.
- 200줄을 썼는데 50줄이면 됐다면, 다시 써라.
- 스스로 물어라: "시니어 엔지니어가 이걸 과하다고 할까?" → 그렇다면 단순화.

## 3. 외과적 변경 — 필요한 것만 건드려라, 네가 어지른 것만 치워라
- 인접한 코드·주석·포매팅을 "개선"하지 마라. 망가지지 않은 걸 리팩터링하지 마라.
- 네 방식과 다르더라도 기존 스타일을 따라라.
- 무관한 데드 코드를 발견하면 언급만 하라 — 지우지 마라.
- 네 변경이 사용하지 않게 만든 import/변수/함수만 제거하라.
- 테스트: 모든 변경 라인이 사용자의 요청으로 직접 추적되어야 한다.

## 4. 목표 주도 실행 — 성공 기준을 정의하라, 검증될 때까지 반복하라
- "검증 추가" → "잘못된 입력 테스트를 쓰고 통과시켜라"
- "버그 수정" → "재현 테스트를 쓰고 통과시켜라"
- "X 리팩터링" → "전후로 테스트가 통과하는지 확인하라"
- 여러 단계 작업이면 간단한 계획을 먼저 말하라: `1. [단계] → 검증: [확인]` …
- 강한 성공 기준은 독립적 반복을 가능하게 한다. "작동하게 해라"는 약한 기준이다.

이 4원칙은 `karpathy-guidelines` 스킬로도 등록되어 있다(코드 작성·리뷰·리팩터링 시 자동 환기).

# 2부 — 작업 방식 (Harness)

이 프로젝트는 Harness 프레임워크로 작업을 단계화한다. 1부 원칙이 각 단계에 그대로 흐른다.

- **`/docs`가 단일 진실 공급원이다.** `PRD.md`(무엇을·왜) / `ARCHITECTURE.md`(구조) / `ADR.md`(기술 결정과 트레이드오프) / `UI_GUIDE.md`(UI 규칙). 작업 전에 읽고, 의도에서 벗어나면 멈춰라.
- **`/harness`** — 탐색 → 논의 → step 설계 → 파일 생성(`phases/<task>/stepN.md`) → `python3 scripts/execute.py <task>` 실행. 절차 상세는 `.claude/commands/harness.md`.
  - step 설계는 1부와 한 몸이다: scope 최소화(=2번), 시그니처 수준 지시(=4번), "X 하지 마라, 이유 Y" 형식의 주의사항(=3번), 실행 가능한 AC 커맨드(=4번).
  - 각 step 파일은 독립 세션에서 실행되므로 자기완결적이어야 한다. "이전 대화에서처럼" 같은 외부 참조 금지 — 필요한 경로·맥락을 전부 파일에 적어라.
- **`/review`** — `/docs` 체크리스트로 변경 사항을 검증. 상세는 `.claude/commands/review.md`.
- **가드레일** — `execute.py`가 이 `CLAUDE.md`와 `/docs/*.md`를 매 step 프롬프트에 주입하고, 완료된 step의 summary를 다음 step에 누적 전달하며, 실패 시 에러를 피드백해 최대 3회 재시도한다.
