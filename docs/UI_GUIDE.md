# UI / UX 가이드 — Totaload ERP

> 이 문서는 `phases/0-mvp/step0` 에서 마스터 계획서 4절(4.1~4.7)을 옮겨 작성했다. 이후 모든 step(특히 step4 frontend)의 가드레일로 주입된다. **제품의 핵심.**

## 4.1 UX 원칙
1. **사람의 검수가 최종 책임 — 시스템은 거든다.** OCR 은 "초안 채우기"일 뿐. 어떤 화면도 "AI 가 했으니 맞다"는 인상을 주지 않는다. OCR 로 채운 값과 빈 값을 시각적으로 구분하고, 등록증 원본을 항상 옆에 *크게·읽을 수 있게* 보여줘 대조하게 한다.
2. **절대 막지 않는다.** OCR 실패·부분·느림 어떤 경우에도 사용자는 직접 입력해 PDF 까지 갈 수 있다. 에러는 길을 막는 게 아니라 다음 행동을 안내한다(빨강 대신 노랑·안내 톤).
3. **긴 대기를 정직하게.** OCR 은 10~90초 걸린다 → 빈 폼을 *즉시* 띄워 아는 값부터 입력하게 하고, 결과가 오면 **빈 칸만** 채운다(사용자가 만진 필드는 절대 덮어쓰지 않음). 진행 상태를 계속 보여주고 "분석 취소" 가능.
4. **하루 수십 건을 빠르게.** Tab 순서 = 등록증 읽는 순서. 명시적 "저장" 부담 없는 자동 저장. 업로드→검수→PDF→다음 건이 최소 클릭. 키보드만으로 전 과정 가능.
5. **일은 사라지지 않는다.** 업로드 응답 즉시 DB 에 draft 레코드 + URL(`/malso/:id`) 생성 → 새로고침/창 닫음 후에도 "작성 중" 목록·검색에서 복구. 미저장(저장 실패 중) 상태에서 이탈 시 경고.
6. **읽을 수 있게 보여준다.** 등록증 이미지 뷰어는 확대/이동/회전 지원(스캔이 작거나 비뚤어도 값을 읽어야 함). 생성한 PDF 는 다운로드/인쇄 전 미리보기.

## 4.2 정보구조 / 내비게이션
- 좌측 고정 사이드바(폭 ~220px): 상단에 제품명 "Totaload", 메뉴 2개 — **말소 입력**, **말소 검색**. 현재 위치 active. 그 외 항목 없음.
- 라우트: `/` → `/malso/new`(말소 입력 시작), `/malso/:id`(특정 차량 입력·검수 — 업로드 후 또는 검색 진입 시), `/malso/search`(검색). 모든 화면 새로고침 안전(상태는 서버).
- `/malso/:id` 와 검색 상세는 같은 화면·같은 컴포넌트(검수=수정=재생성).

## 4.3 말소 입력 / 검수 화면 (`/malso/new` → `/malso/:id`)

**상태 A — 업로드 전 (`/malso/new`)**
- 중앙에 큰 드롭존: "자동차등록증 이미지를 끌어다 놓거나 클릭해서 선택" + 허용 형식·크기 명시(JPG/PNG/PDF, ≤20MB). 드래그 오버 시 하이라이트. 모바일은 `accept="image/*"`+카메라 촬영 허용(best-effort).
- 아래 "작성 중" 목록: 최근 draft 차량(차량번호/차명/시각) — 클릭 시 `/malso/:id` 로 이어 작업. 비어 있으면 안내 문구.
- 클라이언트 1차 검증: 형식/크기 안 맞으면 드롭존 아래 한 줄 안내("HEIC 는 지원하지 않습니다. JPG/PNG/PDF 로 변환해 주세요." 등). 업로드 시작 안 함.

**상태 B — 업로드 직후 · OCR 진행 중 (`/malso/:id`)**
- 즉시 2열: 좌 = 업로드 이미지 뷰어, 우 = **편집 가능한 빈 폼**.
- 우측 폼 상단 배너(파랑): "등록증을 분석하고 있습니다… 결과가 나오면 빈 칸이 자동으로 채워집니다. 그동안 아는 값을 먼저 입력해도 됩니다." + 스피너 + "분석 취소"(→ OCR 없이 수동 입력 모드).
- 사용자가 이 사이 입력한 필드 = OCR 결과로 **덮어쓰지 않음**(만진 필드 집합 추적). OCR 결과는 비어 있는 필드에만 채움.
- 이미지 뷰어: 휠/버튼 확대·축소, 드래그 이동, 90° 회전, "원본 다운로드". PDF 업로드였으면 1페이지 렌더 이미지 + "원본 PDF 보기".

**상태 C — OCR 완료**
- 배너 변화:
  - 성공(ok): 초록 "자동 입력 완료 — 값을 등록증과 대조해 확인하세요." (닫기 가능)
  - 부분(partial): 노랑 "일부 항목만 자동 입력했습니다. 빈 칸을 등록증을 보고 채워 주세요." + 누락 항목 나열(예: "주행거리, 연식").
  - 실패(failed): 노랑(빨강 아님) "자동 입력에 실패했습니다(사유: …). 등록증을 보고 직접 입력해 주세요." + "다시 분석" 버튼.
- 폼 섹션: **소유자**(성명/명칭 · 주민(법인)등록번호 · 주소) / **차량**(자동차등록번호 · 차대번호 · 차명 · 형식·연식 · 주행거리(km) · 차량중량(kg) · 차량총중량(kg)) / **신청서**(작성일 — 기본 오늘). 라벨은 등록증 용어 그대로. Tab 순서 = 이 순서.
- 필드별 시각 단서:
  - OCR 로 채워진 필드: 보통 표시 + 작은 회색 칩 "자동 입력" (수정하면 "수정됨"으로).
  - 비어 있고 중요한 필드(차대번호·차량번호·소유자명 등): amber 좌측 보더 + "확인 필요".
  - 가벼운 형식 안내(차단 아님): 주민번호(앞6-뒤7 숫자 패턴 아니면 안내) · 차대번호(영숫자, 보통 17자) · 차량번호(숫자+한글+숫자) · 숫자필드(콤마 자동 제거, 비숫자면 안내) · 작성일(빈값이면 오늘 자동).
  - 주민등록번호 칸: 기본 마스킹(`8601**-*******`) + 눈 아이콘 토글(저장/PDF 는 평문).
- 자동 저장: 필드 blur 또는 ~0.8초 idle 시 PATCH. 미세 상태 표시("저장 중…/저장됨 HH:MM"). 디바운스 + 마지막 요청만 반영(stale 응답 무시). 저장 실패 시 상단 경고 + 재시도(이때만 이탈 경고).
- 하단 sticky 액션바: 주 버튼 **"말소등록 신청서 PDF 만들기"** → ① 현재 값 저장 → ② ocr-service `/fill-pdf` → ③ 모달에 **PDF 미리보기** + "다운로드" + "인쇄"(브라우저 인쇄, "실제 크기(100%)로 인쇄하세요" 안내) + "닫고 계속 수정" + "새 말소 입력 시작". 중요한 필드가 비었으면 막지 않고 "○○가 비어 있습니다. 그대로 만들까요?" 확인 후 빈 채로 진행(응답 헤더 `X-Missing-Fields` 도 토스트로 안내). 생성 PDF 는 차량에 첨부(이력; 재생성 시 새 항목, 최신 위).
- "다시 분석": 같은 차량에 등록증 뒷면 등 이미지 추가 첨부 가능(첨부만; OCR 재실행은 "다시 분석"으로 명시적). "다시 분석" 시 "현재 입력값을 OCR 결과로 덮어쓸까요?(수동 수정값 포함)" 확인 — 기본은 "빈 칸만 채우기".

## 4.4 말소 검색 화면 (`/malso/search`)
- 상단 큰 검색 입력 — placeholder "차량번호 또는 차대번호 일부 입력". 입력 즉시(디바운스 300ms) 검색, 1자부터. 좌측 돋보기, 우측 ✕(지우기).
- 검색어 비어 있을 때: "최근 차량"(updated_at desc 상위 N개) — 처음 들어와도 쓸모 있게.
- 결과 행: 차량번호 · 차명 · 소유자명 · 상태 배지(작성중/완료) · 등록일. 매칭 부분문자열 하이라이트. 클릭/Enter → `/malso/:id`. **목록·결과에 주민등록번호 노출 안 함.**
- 0건: "‘OOO’에 해당하는 차량이 없습니다." + "말소 입력에서 새로 추가" 링크.
- 너무 많음(>limit): 상위 50개 + "더 정확히 입력하면 범위를 좁힐 수 있습니다." (페이지네이션 없음.)
- 로딩: 행 스켈레톤. 검색 중 이전 결과는 흐리게 유지(깜빡임 방지). 응답 순서 보장(stale 무시).

## 4.5 차량 상세 (`/malso/:id`, 검색에서 진입)
- 좌 = 첨부된 자동차등록증 이미지(들) 뷰어(여러 장이면 썸네일 전환·회전·확대), 우 = 4.3 상태 C 와 동일한 편집 폼(자동 저장) — 즉 검색→상세에서 바로 수정·재생성.
- "첨부 문서" 영역: 등록증 이미지(들) + 생성된 말소등록 신청서 PDF 목록(생성 시각, 최신 위, 각 미리보기/다운로드/인쇄).
- 상단 차량 요약(차량번호 · 차명 · 상태 · 생성/수정 시각) + "← 검색으로". (삭제는 MVP 미포함.)

## 4.6 전역 UX 요소
- **토스트**(우하단, ~3초, 닫기 가능): 저장됨 / PDF 생성됨 / 복사됨 / 에러. 에러 토스트는 가능하면 "재시도" 버튼 포함.
- **버튼 상태**: 비동기 중 비활성 + 인라인 스피너(중복 클릭 방지). 되돌리기 어려운 동작은 확인 모달, Esc 로 닫기.
- **빈 상태**: 모든 목록(작성 중/검색/최근/첨부)에 친절한 문구 + 다음 행동 링크. 중앙정렬은 빈 상태 안내 정도만 예외.
- **로딩**: 즉답 없는 모든 곳에 스피너/스켈레톤. 자동 저장·PATCH 는 낙관적("저장 중" 즉시 표시).
- **에러 표면화**: 비치명적 = 인라인 배너/토스트, 폼 계속 사용 가능. 진짜 못 쓰는 상태(서버·DB 다운)만 전체화면 메시지 + "다시 시도".
- **접근성**: 모든 입력에 `<label>`, 또렷한 포커스 링, 다크 테마 대비 WCAG AA 지향, 키보드만으로 업로드→입력→PDF 가능.
- **반응형**: 데스크톱 2열 우선. <~900px 에선 1열로 쌓되 이미지 뷰어 접기 가능(모바일 전용 최적화는 범위 밖, 깨지지만 않게).
- **표기**: UI 전부 한국어. 날짜 표시 `YYYY. M. D.`(폼 작성일은 PDF용 `YYYY년 M월 D일` 문자열로 저장). 숫자 천단위 콤마는 *표시*에만(저장·PDF 는 콤마 없음).
- **카피 톤**: 담백·행동지향. "오류가 발생했습니다" 대신 "등록증을 다시 분석해 보세요" 식. 비난 금지.

## 4.7 비주얼 토큰

"매일 쓰는 도구" 톤. **AI 슬롭 안티패턴 금지** — 아래는 절대 쓰지 않는다:

| 금지 사항 | 이유 |
|---|---|
| backdrop-filter: blur() (글래스 모피즘) | AI 템플릿의 가장 흔한 징후 |
| 배경 그라데이션 텍스트 (gradient-text) | AI 가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 배지 | 장식. 사용자에게 가치 없음 |
| box-shadow 글로우 애니메이션 | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 rounded-2xl | 균일한 둥근 모서리 = 템플릿 느낌 |
| 배경 gradient orb (blur-3xl 원형) | 모든 AI 랜딩 페이지의 장식 |

### 색상
| 용도 | 값 |
|---|---|
| 페이지 배경 | `#0a0a0a` |
| 카드 배경 | `#141414` |
| 보더 | `neutral-800` |
| 주 텍스트 | `text-white` |
| 본문 | `text-neutral-300` |
| 보조 | `text-neutral-400` |
| 비활성 | `text-neutral-500` |
| 성공 | green (`#22c55e` 계열) |
| 에러 | red (`#ef4444` 계열) |
| 주의(빈/확인필요 필드·partial/failed 배너) | amber |
| 링크/active 포인트 | 절제된 포인트색 — **보라 금지** |

### 컴포넌트
- 카드: `rounded-lg border border-neutral-800 bg-[#141414] p-6`
- 입력: `rounded-md bg-neutral-900 border border-neutral-800 px-3 py-2 focus:border-neutral-500`
- 버튼 Primary: `rounded-md bg-white text-black hover:bg-neutral-200 disabled:opacity-50`
- 버튼 Secondary/Text: `text-neutral-400 hover:text-neutral-200`
- 상태 배지: 작은 알약 — 작성중 = neutral, 완료 = green
- (모든 카드가 동일 radius 가 되지 않게 — 표·리스트·모달은 radius 를 달리 쓸 수 있다.)

### 레이아웃
- 사이드바 220px + 콘텐츠(좌측 정렬). 2열은 폼/뷰어 각 minmax.
- 간격: gap-3~4, 섹션 간 space-y-6~8.
- 정렬: 좌측 정렬 기본. 중앙 정렬은 빈 상태 안내 정도만 예외.

### 타이포그래피
| 용도 | 스타일 |
|---|---|
| 페이지 제목 | `text-2xl font-semibold` |
| 섹션 제목 | `text-sm font-medium text-neutral-400 uppercase tracking-wide` |
| 본문 | `text-sm text-neutral-300` |
| 라벨 | `text-xs text-neutral-400` |

### 애니메이션
- fade-in(~0.2s), 토스트 slide-in 정도만. 그 외 모든 애니메이션 금지(특히 글로우/펄스).

### 아이콘
- 인라인 SVG, strokeWidth 1.5. 아이콘을 박스 컨테이너(둥근 배경 박스)로 감싸지 않는다.
